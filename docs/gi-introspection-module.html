3.1 The (gi introspection) module

The (gi introspection) module contains procedures that dynamically create Guile interfaces for GObject-based libraries. The interfaces are created by examining the information in their typelib files. A typelib file contains information on the public classes, functions, and variables in a GObject-based library. Rather than using filenames, namespace strings are used to refer to typelib files. Some well known namespace strings are “Gtk” and “Glib”.

A minimal example for this library would be to parse a typelib file using the require-version procedure, and then generate the bindings using the, import procedure, as in the following example. The autogenerated interface will be loaded into the current module. 

(use-modules (gi introspection))
(require-version "Gtk" "3.0")
(import "Gtk")
Procedure: require-version namespace version

Given namespace, a string like “Gtk”, and version, a string containing a version like “3.0”, require-version ensures the typelib file for this namespace get loaded with the given version. If the namespace was already loaded with a different version or if a different version was required previously, it will raise an error.

The return value is unspecified.

Procedure: import namespace

Given namespace, a string like “Gtk”, this procedure autogenerates bindings to the library and loads them into the current module. Note that the namespace must already be loaded by using the require-version procedure, or an error will be thrown.

The return value is unspecified.

Procedure: require-foreign namespace

Given namespace, a string of the form “cairo”, require-foreign ensures that the interface for a given library gets loaded. Unlike the require-version procedure, which loads interfaces from GObject typelib files, this procedure loads interfaces coded into Guile GI itself. These interfaces are created when Guile GI is compiled, and cannot be modified later without recompiling Guile GI.

In practice, only one such interface exists. It is the interface for the Cairo drawing library.

It will throw an error if the given interface does not exist. The return value is unspecified.

Procedure: check-version namespace version

Given namespace, a string like “Gtk”, and version, a string containing a version like “3.0”, check-version will throw an error if the given namespace was not already loaded using require-version. Otherwise, it does nothing.

The return value is unspecified.

Procedure: get-required-version namespace

Given namespace, a string like “Gtk”, this procedure returns, as a string, the version of any namespace previously loaded by require-version. If the given namespace was not previously loaded, it returns #f.

Variable: version-info

This variable is set to a list of 3 integers that represent the version of Guile GI.
