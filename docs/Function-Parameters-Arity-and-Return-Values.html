2.2 Function Parameters, Arity, and Return Values

When using Guile-GI, the Scheme versions of introspected C functions and callbacks will not always have the same number of input, output, and return parameters as the C functions do. The GObject Introspection information designates the parameters to C functions as either in, in/out or out parameters. The in/out and out parameters are C pointers, so that the C function may return data by reference. To understand how they will appear in the Scheme mapping, a few rules must be understood.

The basic rules as follows:

  • If the C function has an input parameter, the Scheme binding will have an input parameter.
  • If the C function has a return value that is not void, the Scheme binding will return a value.
  • If the C function has an in/out parameter, the Scheme binding will take an input parameter and return the output value.
  • If the C function has an out parameter, the Scheme binding will not have it as an input parameter but will return it as an output value.

By these rules, if the C function returns void and has no out or in/out parameters, the return value of the Scheme procedure is *unspecified*.

If a C function returns a non-void value and has zero out or in/out parameters, or if a C function returns void and has a single out or in/out parameter, the Scheme binding will return a value.

Otherwise, the Scheme binding will return multiple values, as with the Guile procedures values and receive.
