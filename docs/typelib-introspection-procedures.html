3.1 Typelib introspection procedures

A GObject typelib file contains information on the public classes, functions, and variables in a GObject-based library, and most GObject-complient libraries, such as GLib and GTK, provide one. Usually these files are on the filesystem in a single directory location. On Fedora, for example, these files may be stored in /var/lib64/girepository-1.0.

Procedure: get-typelib-search-path

This returns, as a list of strings, the current search path used to find typelib files.

If it returns an empty list, this indicates that it is only searching the default typelib directory.

Procedure: prepend-typelib-search-path directory

Given directory, a string representation of a filesystem directory, this prepends that directory to the typelib search path.

Procedure: export-typelib namespace version

Given namespace, a string that names a typelib library, such as “Gtk” or “GLib”, and version, a string that names a version of a typelib library, like “2.0” or “3.0”, this procedure parses a typelib file and generates the Scheme source code for a binding to the library. The source code is returned as a string, which you should save to a file.

The procedure may throw an error if a typelib file corresponding to namespace and version cannot be found.

Procedure: import-typelib namespace version

Given namespace, a string that names a typelib library, such as “Gtk” or “GLib”, and version, a string that names a version of a typelib library, like “2.0” or “3.0”, this procedure parses a typelib file and generates procedures, constants, and types which are loaded into the current environment.

The procedure may throw an error if a typelib file corresponding to namespace and version cannot be found.

This procedure implicitly calls load-typelib.

The return value is unspecified.

Procedure: load-typelib namespace version

Given namespace, a string that names a typelib library, such as “Gtk” or “GLib”, and version a string that names a version of a typelib library , like “2.0” or “3.0”, this procedure reads a typelib file, and indexes the information in the typelib file into several hash tables. The hash tables are created in the current environment, and have names such as %gi-constants and %gi-functions.

The procedure may throw an error if a typelib file corresponding to namespace and version cannot be found.

The return value is unspecified.

Variable: %gi-constants
Variable: %gi-enums
Variable: %gi-flags
Variable: %gi-functions
Variable: %gi-methods
Variable: %gi-callbacks
Variable: %gi-structs
Variable: %gi-unions
Variable: %gi-objects
Variable: %gi-interfaces
Variable: %gi-signals

These are all hash tables generated by load-typelib. They contain information parsed from a typelib file and indexed for easier access by procedures such as gi-constant-value and gi-function-invoke.

gi-constant-value: name
gi-flag-value: category name
gi-enum-value: category name
gi-struct-ref: ptr type-name field-name
gi-struct-set: ptr type-name field-name value
gi-function-invoke: name [args...]
gi-method-prepare: name [args...]
gi-method-send: object list-of-args
gi-lookup-type: name
gi-lookup-callback-info: name

These procedures are used to generate the bindings. They lookup introspection information stored in hash tables generated by the load-typelib procedure and convert between C and Guile types.

They are used in the procedures generated by export-typelib and import-typelib.
