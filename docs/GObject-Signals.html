3.5 GObject Signals

GObjects have a functionality for defining and calling callback procedures that it calls signals. Note that these signals have nothing to do with Unix signals like SIGTERM.

To hook a callback procedure to an object’s signal, use the signal-connect procedure. Consider the following example. 

(define (activate app)
   ...
   )

(signal-connect app         ; A GObject
                "activate"  ; The name of a signal it might emit
                activate)   ; A procedure to call when emitted

In that example, the caller is attaching a procedure named activate to the object app’s “activate” signal. When the gobject object calls all the callback procedures attached to its “activate” signal it will call this activate procedure.

When a callback procedure is called it may receives some arguments from the caller and some from user data. In the example, activate is supposed to take a single argument. This argument – app – is supplied by the caller, as are all arguments to signals.

Previous versions added a second (in the general case last) argument analogous to the user-data argument used excessively within GObject. Such user data parameters have been removed as they are unnecessary when using (anonymous) procedures in Scheme.

When you are connecting many signals of one object, you may wish to save some typing by doing the following: 

(modify-signals object
  (connect signal-a handler-a)
  (connect signal-b handler-b)
  ...
  (connect signal-z handler-z))

This syntax roughly expands to 

(begin
  (signal-connect object ``signal-a'' handler-a)
  (signal-connect object ``signal-b'' handler-b)
  ...
  (signal-connect object ``signal-z'' handler-z)))

except that object is temporarily bound to an unique identifier if is not already one. You could for instance fetch your object from a (tree of) the resulting object would be active for the scope of the signal modifications and the reference to it discarded afterwards.

Signals can also be connected after the main handler. To do so, supply #t as the optional fourth argument to signal-connect or use connect-after in modify-signals.
