3.4 GObject Methods

The GObject system has its own functionality for methods. It can have methods for GObject-based structs, unions, and objects. To call a GObject method, use the with-object syntax or the method-call procedure.

As an example, the following two calls are equivalent. 

(with-object window (set-title "MAIN WINDOW"))
(call-method window "set-title" '("MAIN WINDOW"))
Syntax: with-object object method ...

Given a GObject object, this syntax attempts to find a match between the object and the given method(s). If the object’s type or any of its parent types have the specified method(s), they will be called. If some method does not exist, an error will be thrown.

Each individual method is a list with the first element being a symbol, which names the method to be called and the remaining arguments being the arguments to the call.

The with-object syntax is built using the call-method procedure.

Procedure: call-method object method-name list-of-args

Given a GObject object and a string method-name, this procedure searches for a method of the given name associated with this object’s type, or, failing that, with any of this object’s parents’ types. If an associated method is found, it is called with the given list of arguments.

If no associated method is found, or if the list of arguments doesn’t match the expected argument types or count, an error is thrown.
