3.3 GType

A <GType> is a Guile type that wraps GObject GType. Basically it is an integer ID that GObject uses to identify a type. It is a type that holds a type that IDs a type.

GObject has its own custom class structure. The following procedures provide information about how a <GType> forms part of that class structure.

Procedure: ->gtype object

Given object, a GObject struct, union, or object, this procedure returns its GObject type as a <GType>.

If the input parameter is an integer, this procedure will return a <GType> if that integer is a GType ID, or #f if it is not.

If the input parameter is a string, the procedure will return a <GType> if that string happens to be the name of a type, or #f if it is not.

Procedure: gtype-get-scheme-type gtype

Given gtype of type <GType>, this procedure will return the Scheme type that holds instances of that type.

Procedure: gtype-get-name gtype

Given gtype of type <GType>, this procedure returns a string with GObject’s internal name for this type. This may differ slightly from the name that Guile-GI gives to the type.

Procedure: gtype-get-parent gtype
Procedure: gtype-get-children gtype

These procedures return the parent type and the child types of a specified <GType>.

Procedure: gtype-is-a? object type

Returns #t of object is an instance of a type.
